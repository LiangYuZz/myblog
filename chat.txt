  ### 我看出来的几个关键点

  #### 1. 这玩意已经不是“小脚本”了

  它现在其实是个 Hugo 本地后台，而且比最初版本大很多，功能已经扩成一套工作流工具了。

  #### 2. 真正的薄弱点不在 GUI，在 front matter 解析

  parse_markdown() 是手搓的，比较脆：

  - 多行 description
  - 带冒号的值
  - 复杂 YAML
  - 嵌套字段
  - TOML front matter

  这些都容易解析歪。

  #### 3. 预览不是真正的 Hugo 预览

  现在的预览是：

  - 一个 Tk 文本预览
  - 一个 Python markdown 转 HTML 的浏览器预览

  它不是 Hugo/主题真实渲染，所以如果你用了：

  - shortcode
  - 主题模板
  - taxonomy 页
  - 主题样式联动

  它预览会和最终站点不完全一致。

  #### 4. Ctrl+D 现在已经能走通

  因为我已经把 updateblog.bat 修好了，所以你这个 GUI 里的“上传博客”按钮现在应该能正常用了。
